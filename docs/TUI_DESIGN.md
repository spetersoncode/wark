# Wark: TUI Design

> Terminal User Interface specification for interactive task management

## 1. Overview

The wark TUI provides a rich, interactive interface for managing tickets, viewing status, and responding to agent requests. Built with [Bubble Tea](https://github.com/charmbracelet/bubbletea), it offers a vim-like experience with keyboard-driven navigation.

## 4. Global Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `1`-`5` | Switch views |
| `?` | Show help |
| `q` | Quit (with confirmation if changes pending) |
| `/` | Global search |
| `p` | Switch project |
| `n` | Create new ticket |
| `r` | Refresh data |
| `Tab` | Next pane/section |
| `Shift+Tab` | Previous pane/section |
| `Esc` | Close modal/cancel |

## 5. Color Scheme

| Element | Color | Hex |
|---------|-------|-----|
| Background | Dark gray | `#1a1a2e` |
| Foreground | Light gray | `#eaeaea` |
| Accent | Cyan | `#00d4ff` |
| Success/Done | Green | `#00ff88` |
| Warning | Yellow | `#ffcc00` |
| Error/Blocked | Red | `#ff4757` |
| Working | Blue | `#3742fa` |
| Muted | Gray | `#666666` |

**Priority colors:**
- Highest: Red (`#ff4757`)
- High: Orange (`#ffa502`)
- Medium: Yellow (`#ffcc00`)
- Low: Green (`#00ff88`)
- Lowest: Gray (`#666666`)

## 6. Responsive Behavior

### Minimum Terminal Size
- Width: 80 columns
- Height: 24 rows

### Adaptive Layout

**< 100 columns:**
- Board view shows 2 columns instead of 4
- Horizontal scroll enabled

**< 120 columns:**
- Board view shows 3 columns

**>= 120 columns:**
- Full 4-column board view

## 7. Accessibility

- All colors have sufficient contrast ratio (>4.5:1)
- All actions available via keyboard
- Screen reader compatible status announcements
- No reliance on color alone (icons/symbols supplement)

## 8. State Persistence

The TUI remembers:
- Last active view
- Filter settings per view
- Sort preferences
- Expanded/collapsed states
- Window position (if supported by terminal)

Stored in: `~/.wark/tui_state.json`
